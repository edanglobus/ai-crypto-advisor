# Deployment Guide

Production setup: **frontend on Vercel**, **backend + PostgreSQL on Render**.
Auth uses cross-site `httpOnly` cookies (the backend already sends
`SameSite=None; Secure` in production and trusts the Render TLS proxy).

Deploy the **backend first** (you need its URL for the frontend), then the
frontend, then point them at each other.

---

## 1. Backend + database — Render

Uses [`render.yaml`](../render.yaml) (a Blueprint that provisions the web service
**and** a free Postgres instance).

1. Push the repo to GitHub (done).
2. Render Dashboard → **New → Blueprint** → select this repo → **Apply**.
   - It creates `ai-crypto-advisor-api` (web service, `rootDir: backend`) and
     `ai-crypto-advisor-db` (Postgres).
   - `DATABASE_URL` is wired automatically; `JWT_*` secrets are auto-generated.
   - Build: `npm install && npx prisma generate && npm run build`
   - Start: `npx prisma migrate deploy && node dist/server.js` (runs migrations
     on every deploy).
3. After the first deploy, open the service → **Environment** and set:
   - `FRONTEND_URL` = your Vercel URL (fill in after step 2; e.g.
     `https://ai-crypto-advisor.vercel.app`).
   - Optional API keys: `OPENROUTER_API_KEY`, `CRYPTOPANIC_API_KEY`,
     `COINGECKO_API_KEY` (the app falls back gracefully if blank).
4. Note the backend URL, e.g. `https://ai-crypto-advisor-api.onrender.com`.

> Health check: `GET /health`. First request after idle may be slow (free tier
> cold start).

## 2. Frontend — Vercel

Uses [`frontend/vercel.json`](../frontend/vercel.json) (Vite + SPA rewrites so
client-side routes work on refresh).

1. Vercel → **Add New → Project** → import this repo.
2. Set **Root Directory** = `frontend`.
3. Add environment variable:
   - `VITE_API_URL` = `https://<your-render-backend>.onrender.com/api`
4. Deploy. Note the URL (e.g. `https://ai-crypto-advisor.vercel.app`).

## 3. Connect them

1. Back on Render, set `FRONTEND_URL` to the exact Vercel URL → save (redeploys).
   This must match for CORS (`credentials: true`) and cookies to work.
2. Re-deploy the frontend if you changed `VITE_API_URL` (it's baked in at build).

## 4. Verify

- Open the Vercel URL → sign up → onboarding → dashboard loads with live data.
- DevTools → Network: API calls to the Render URL return `200` with
  `Set-Cookie` (Secure, SameSite=None) and `Access-Control-Allow-Credentials: true`.
- Refreshing a deep link (e.g. `/settings`) still loads (SPA rewrite working).

## Notes & gotchas

- **Cookies are cross-site** (different domains). Modern browsers accept
  `SameSite=None; Secure` first-party XHR cookies; if you ever hit third-party
  cookie blocking, the alternative is to proxy `/api` through Vercel rewrites so
  both share one origin.
- **Migrations** run automatically on deploy via `prisma migrate deploy`.
- **Secrets**: never commit `.env`; all secrets live in the Render/Vercel
  dashboards. `JWT_*` are auto-generated by the blueprint.
- **DB access** (deliverable): from the Render Postgres page use the external
  connection string with `psql`/TablePlus, or `npx prisma studio` locally
  pointed at it.
